This script queries DevRev for nitpick-tagged issues in "triage" stage. It outputs one issue display_id per line for the runner to process.

Script structure:
```bash
#!/usr/bin/env bash
# scripts/circleci/query_nitpick_issues.sh
# Queries DevRev API for nitpick-tagged issues pending agent processing.
# Outputs one display_id per line. Used by run_nitpick_fix_agent.sh.
set -euo pipefail

# --- Validate environment ---
: "${DEVREV_PAT:?'DEVREV_PAT must be set (from CircleCI devrev-api context)'}"
DEVREV_API_BASE="https://api.devrev.ai"

# --- Step 1: Resolve "nitpicked" tag ID ---
TAG_RESPONSE=$(curl -sf -X POST "${DEVREV_API_BASE}/tags.list" \
  -H "Authorization: ${DEVREV_PAT}" \
  -H "Content-Type: application/json" \
  -d '{"name": ["nitpicked"]}')

TAG_ID=$(echo "${TAG_RESPONSE}" | jq -r '.tags[0].id // empty')
if [ -z "${TAG_ID}" ]; then
  echo "ERROR: 'nitpicked' tag not found in DevRev" >&2
  exit 1
fi

# --- Step 2: Query issues with that tag in triage stage ---
ISSUES_RESPONSE=$(curl -sf -X POST "${DEVREV_API_BASE}/works.list" \
  -H "Authorization: ${DEVREV_PAT}" \
  -H "Content-Type: application/json" \
  -d "{
    \"type\": [\"issue\"],
    \"tags\": [\"${TAG_ID}\"],
    \"stage.name\": [\"triage\"]
  }")

# --- Step 3: Extract display_ids ---
ISSUE_IDS=$(echo "${ISSUES_RESPONSE}" | jq -r '.works[].display_id // empty')

if [ -z "${ISSUE_IDS}" ]; then
  echo "No nitpick-tagged issues in triage stage." >&2
  exit 0
fi

echo "${ISSUE_IDS}"
```

Create `scripts/circleci/run_nitpick_fix_agent.sh` in devrev-web (`/Users/kavinash/arcade/test/devrev-web/scripts/circleci/run_nitpick_fix_agent.sh`).

This is the main agent lifecycle script. It:
1. Validates required environment variables
2. Runs the concurrency guard (per D-15, CICD-02)
3. Fetches full issue metadata from DevRev
4. Prepares the workspace (branch creation per D-17)
5. Injects issue context into prompt template via envsubst
6. Invokes `claude -p` with full sandboxing (per D-01, D-04, D-05, D-14)
7. Runs post-run guardrails (file existence, scope, branch naming)
8. Logs metrics

Full script content:

```bash
#!/usr/bin/env bash
# scripts/circleci/run_nitpick_fix_agent.sh
# Orchestrates the nitpick fix agent lifecycle on CircleCI.
# Adapted from devrev/mobius:scripts/circleci/run_daily_tech_debt_agent.sh
set -euo pipefail

# ============================================================
# Configuration
# ============================================================
: "${NITPICK_ISSUE_ID:?'NITPICK_ISSUE_ID must be set'}"
: "${DEVREV_PAT:?'DEVREV_PAT must be set (from CircleCI devrev-api context)'}"

PROMPT_FILE="scripts/circleci/nitpick_fix_prompt.md"
MAX_TURNS=65
MAX_BUDGET=5.00
DEVREV_API_BASE="https://api.devrev.ai"

# ============================================================
# Concurrency Guard (CICD-02)
# ============================================================
BRANCH_NAME="nitpick/${NITPICK_ISSUE_ID}-fix"

if git ls-remote --heads origin "${BRANCH_NAME}" 2>/dev/null | grep -q "${BRANCH_NAME}"; then
  EXISTING_PR=$(gh pr list --head "${BRANCH_NAME}" --state open --json number -q '.[0].number' 2>/dev/null || echo "")
  if [ -n "${EXISTING_PR}" ]; then
    echo "PR #${EXISTING_PR} already open for ${NITPICK_ISSUE_ID}. Skipping."
    exit 0
  fi
fi

# ============================================================
# Install Claude Code CLI (AGNT-01)
# ============================================================
echo "Installing Claude Code CLI..."
npm install -g @anthropic-ai/claude-code

# ============================================================
# Bedrock Authentication (D-01)
# ============================================================
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION="${AWS_REGION:-us-east-1}"
export ANTHROPIC_DEFAULT_SONNET_MODEL='us.anthropic.claude-sonnet-4-6'

# ============================================================
# Fetch Issue Metadata
# ============================================================
echo "Fetching issue ${NITPICK_ISSUE_ID} from DevRev..."
ISSUE_DATA=$(curl -sf -X GET \
  "${DEVREV_API_BASE}/works.get?id=${NITPICK_ISSUE_ID}" \
  -H "Authorization: ${DEVREV_PAT}")

TITLE=$(echo "${ISSUE_DATA}" | jq -r '.work.title // ""')
DESCRIPTION=$(echo "${ISSUE_DATA}" | jq -r '.work.body // ""')
METADATA_JSON=$(echo "${ISSUE_DATA}" | jq -c '.work | {display_id, title, body, priority, priority_v2, tags, artifacts, applies_to_part}')

if [ -z "${TITLE}" ] || [ "${TITLE}" = "null" ]; then
  echo "ERROR: Could not fetch issue ${NITPICK_ISSUE_ID} from DevRev" >&2
  exit 1
fi

echo "Issue: ${TITLE}"

# ============================================================
# Prepare Workspace (D-17, CICD-03)
# ============================================================
git config --global user.name 'devrev-ci-bot'
git config --global user.email 'ci-bot@devrev.ai'
git checkout -b "${BRANCH_NAME}" 2>/dev/null || git checkout "${BRANCH_NAME}"

# ============================================================
# Inject Issue Context into Prompt Template
# ============================================================
export NITPICK_ISSUE_ID TITLE DESCRIPTION METADATA_JSON
PROMPT=$(envsubst '${NITPICK_ISSUE_ID} ${TITLE} ${DESCRIPTION} ${METADATA_JSON}' < "${PROMPT_FILE}")

# ============================================================
# Run Agent (AGNT-01, AGNT-04, D-04, D-05, D-14)
# ============================================================
echo "Starting nitpick fix agent for ${NITPICK_ISSUE_ID}..."
set +e
claude -p "${PROMPT}" \
  --model sonnet \
  --max-turns "${MAX_TURNS}" \
  --max-budget-usd "${MAX_BUDGET}" \
  --permission-mode bypassPermissions \
  --allowedTools \
    "Read" "Grep" "Glob" "Edit" "Write" \
    "Bash(pnpm run workspace-lint *)" \
    "Bash(pnpm run affected:typecheck *)" \
    "Bash(pnpm install *)" \
    "Bash(git add *)" "Bash(git commit *)" "Bash(git push *)" \
    "Bash(git diff *)" "Bash(git log *)" "Bash(git status *)" \
    "Bash(git checkout *)" "Bash(git branch *)" \
    "Bash(gh pr create *)" "Bash(gh pr view *)" \
    "Bash(nx affected *)" "Bash(npx *)" \
  --output-format json \
  --verbose \
  2>&1 | tee /tmp/agent-output.json
AGENT_EXIT=$?
set -e

# ============================================================
# Post-Run Guardrails
# ============================================================
echo "Running post-run guardrails..."

# Parse agent output
if [ -f /tmp/agent-output.json ]; then
  COST=$(tail -1 /tmp/agent-output.json | jq -r '.total_cost_usd // 0' 2>/dev/null || echo "0")
  TURNS=$(tail -1 /tmp/agent-output.json | jq -r '.turns // 0' 2>/dev/null || echo "0")
else
  COST=0
  TURNS=0
fi

# Guardrail 1: Branch naming validation
CURRENT_BRANCH=$(git branch --show-current)
if ! echo "${CURRENT_BRANCH}" | grep -qE '^nitpick/[A-Z]+-[0-9]+-'; then
  echo "GUARDRAIL FAIL: Branch '${CURRENT_BRANCH}' does not match nitpick/ISS-XXXX-* pattern" >&2
  exit 1
fi

# Guardrail 2: File existence (hallucination detection)
HALLUCINATED=0
for FILE in $(git diff --name-only origin/main...HEAD 2>/dev/null); do
  if [ ! -f "${FILE}" ]; then
    echo "GUARDRAIL FAIL: Modified file does not exist: ${FILE}" >&2
    HALLUCINATED=1
  fi
done
if [ "${HALLUCINATED}" -eq 1 ]; then
  echo "Aborting -- hallucinated file paths detected" >&2
  exit 1
fi

# Guardrail 3: Scope check (blast radius)
PACKAGES_TOUCHED=$(git diff --name-only origin/main...HEAD 2>/dev/null | \
  grep -oE '^(libs|apps)/[^/]+' | sort -u | wc -l | tr -d ' ')
if [ "${PACKAGES_TOUCHED}" -gt 3 ]; then
  echo "GUARDRAIL WARNING: Agent touched ${PACKAGES_TOUCHED} packages -- manual review required"
fi

# ============================================================
# Metrics Logging
# ============================================================
echo "$(date -u +%Y-%m-%dT%H:%M:%SZ),${NITPICK_ISSUE_ID},${AGENT_EXIT},${COST},${TURNS},${PACKAGES_TOUCHED}" \
  >> /tmp/nitpick-agent-metrics.csv

# ============================================================
# Final Status
# ============================================================
if [ "${AGENT_EXIT}" -ne 0 ]; then
  echo "Agent failed with exit code ${AGENT_EXIT} after ${TURNS} turns (cost: \$${COST})"
  exit 1
fi

echo "Agent completed successfully: ${TURNS} turns, \$${COST} cost, ${PACKAGES_TOUCHED} packages touched"
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI Runner -> Bedrock API | AWS credentials cross from CircleCI context to Bedrock; secrets in env vars |
| CI Runner -> DevRev API | DEVREV_PAT crosses from CircleCI context to DevRev; secret in env var |
| Agent -> Repository | Agent has write access to repo via allowed git/gh tools |
| DevRev Issue -> Agent Prompt | Untrusted user-created issue content injected into agent prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | run_nitpick_fix_agent.sh | mitigate | Never echo DEVREV_PAT or AWS creds; use `set +x` before sections handling secrets; pipe agent output to file not stdout for secrets |
| T-04-02 | Elevation of Privilege | --allowedTools flag | mitigate | Explicit allowlist with space-before-asterisk pattern (e.g., `Bash(git diff *)` not `Bash(git diff*)`); no `Bash(curl:*)` or `Bash(rm:*)` |
| T-04-03 | Tampering | DevRev issue content -> prompt | mitigate | envsubst only substitutes declared variables (not arbitrary shell expansion); issue content is injected as data in the prompt, not as shell commands; no `eval` on issue content |
| T-04-04 | Denial of Service | Agent loop | mitigate | --max-turns 65 and --max-budget-usd 5.00 hard caps; CircleCI no_output_timeout: 30m kills stalled jobs |
| T-04-05 | Spoofing | Concurrency guard | accept | Branch existence check is a soft lock; race condition window is small (seconds between check and branch creation); acceptable for hourly cron volume |
| T-04-06 | Information Disclosure | Agent output JSON logged | mitigate | Agent output goes to /tmp/agent-output.json (ephemeral CI workspace); metrics CSV logs only issue ID, cost, turns -- no secrets |
</threat_model>
